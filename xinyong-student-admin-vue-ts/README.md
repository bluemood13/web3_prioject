# 用户信用评分算法说明文档

## 1. 概述

本系统根据用户的历史交易记录、资产持有量以及盈利能力，综合计算用户的信用评分。评分旨在反映用户在平台上的活跃度、资金实力及投资水平。

## 2. 评分维度

系统基础分为 **400分**，总分上限为 **950分**。

### 2.1 交易活跃度 (Activity)

反映用户的交易频次，鼓励用户多参与市场交易。

- **算法**：`交易总笔数 × 5`
- **上限**：200分
- **说明**：每进行一次买入或卖出操作，信用分增加5分，最高累计加200分（即40笔交易）。

### 2.2 资产规模 (Assets)

反映用户的持币数量，代表用户的资金留存意愿和实力。

- **算法**：`持有代币数量 × 10`
- **上限**：150分
- **说明**：每持有1个单位的代币，信用分增加10分，最高累计加150分。

### 2.3 盈利能力 (Profitability)

反映用户的投资水平和“低买高卖”的能力，高盈利能力意味着更理性的投资行为。

- **策略 A (均价盈利)**：
  - 如果 `平均卖出价格 > 平均买入价格`
  - **加分**：+100分
- **策略 B (总额盈利)**：
  - 如果 `总卖出收入 > 总买入成本`
  - **加分**：+50分

## 3. 评分计算公式

```javascript
总分 =
  基础分(400) +
  MIN(交易次数 * 5, 200) +
  MIN(持有代币数 * 10, 150) +
  (卖出均价 > 买入均价 ? 100 : 0) +
  (总卖出额 > 总买入额 ? 50 : 0);
```

> **注意**：最终得分若超过 950 分，则按 950 分计算。

## 4. 信用等级对照表

| 分数区间  | 信用等级 | 颜色标识 |
| :-------- | :------- | :------- |
| ≥ 800     | 信用极好 | 🟢 绿色  |
| 700 - 799 | 信用优秀 | 🟢 绿色  |
| 600 - 699 | 信用良好 | 🔵 蓝色  |
| 500 - 599 | 信用一般 | 🟡 黄色  |
| < 500     | 信用较差 | 🔴 红色  |

## 5. 核心代码逻辑

该逻辑位于 `src/views/xinyong/myUserQianbao/index.vue` 的 `analyzeCredit` 函数中。

1.  **获取数据**：调用 `api.xinyong.jiaoyi.getJiaoyiListAll` 获取全量交易记录。
2.  **统计遍历**：计算总买入量、总买入成本、总卖出量、总卖出收入，并得出买卖均价。
3.  **计算得分**：应用上述公式进行累加。
4.  **结果展示**：通过弹窗展示最终得分及买卖均价对比。
